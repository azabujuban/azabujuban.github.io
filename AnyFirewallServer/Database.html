      AnyFirewallServer: Database

      Friday, November 14, 2014 5:37 PM

      This section describes how the Eyeball AnyFirewall™ Server uses the database and how to setup new accounts. The database tables can be created using the database script included in the Eyeball AnyFirewall™ server package. This script will also create a few test accounts, which can be used to test the server.

      Administrators only need to access the tables required for provisioning and statistics. All other tables are required for internal purposes only and should not be modified.

      Note: Provisioning and gathering of statistics is also available through the Eyeball Server Management application.

      Provisioning

      Adding and removing user accounts requires accessing the account table and the afwaccount table in the database. The account table is used for user authentication by all of Eyeball's servers, and the afwaccount table contains AnyFirewall™ specific settings for each user.

      The account table has the following columns:

      Name Type
      account_id unsigned auto_increment
      user_id varchar(32)
      Password varchar(32)
      Active varchar(1)
      Created datetime

      Note: In order to add a new user, the user's ID (the name of the user, e.g., eyeball) and the password must be added to the account table.

      The afwaccount table includes an afwstatus column, which, if set to zero, disables a user from using the AnyFirewall™ server(s). This allows an administrator to disable a user from using this service only.

      Statistics

      The Eyeball AnyFirewall™ Server periodically logs statistics and usage information to the database. In addition, each user's activity, e.g., allocating server ports, is written to the database when such events occur. The information can be extracted from the afwstatistics, afwsummary, afwuserstatistics, and afwallocation tables.

      Database tables

      This section describes and summarizes all the database tables used by the Eyeball AnyFirewall™ Server. The access mode of each table is also specified. The fields mentioned are required for the proper operation of the server.

      account

      This table is used by all Eyeball server products for user authentication. password is the password of the user in encrypted format. The encrypted passwords are generated by the pass3des tool supplied with the server package. The active column specifies whether the user account is active (Yes or No), or whether it is Abused (A). An abused account means that the user account has been abused and is disabled, resulting in a failed login for the user.

      CREATE TABLE `account` (
  `account_id` int(10) unsigned NOT NULL auto_increment,
  `user_id` varchar(32) NOT NULL default ' ',
  `password` varchar(32) NOT NULL default ' ',
  `active` varchar(1) NOT NULL default 'Y',
  `created` datetime NOT NULL default '1970-01-01 00:00:00',
  PRIMARY KEY  (`account_id`),
  UNIQUE KEY `account_user_index_idx` (`user_id`)
)
      Column Type Description
      account_id integer(10) A unique identifier for the user account
      user_id varchar(32) The user ID associated with this account
      password varchar(32) The password used to access the account (this is encrypted when stored)
      active varchar(1) Indicates whether the account is active (Y), not inactive (N), or abused (A)
      created datetime The date/time the account was created

      afwaccount

      This table contains the specific permissions of a user for the Eyeball AnyFirewall™ Server. If the user has a valid account in the account table, and the user has an entry in this table, with afwstatus set to 1, then that user has permission to use the AnyFirewall™ Server. Additionally, this table specifies whether the users' calls are recorded and whether or not the user has a user specific maximum bandwidth per allocation. If maxbandwidth is set to 0 or NULL, then the system default value will be used for the user.

      CREATE TABLE `afwaccount` (
  `account_id` int(10) unsigned NOT NULL default '0',
  `wiretap` int(11) NOT NULL default '0',
  `afwstatus` int(11) NOT NULL default '1',
  `maxbandwidth` int(10) unsigned default NULL,
  PRIMARY KEY  (`account_id`)
)
      Column Type Description
      account_id integer(10) A unique identifier for the user account
      wiretap integer(1) Indicates whether the user's calls are recorded (0 = no)
      afwstatus integer(1) Indicates whether the user has access to the AnyFirewall Server (1 = yes)
      maxbandwidth integer(10) The maximum bandwidth the user can be allocated. If this is set to 0, the system uses the default bandwidth

      afwallocation

      All allocations from a server are stored in this table. An entry in the table is created when a new port is allocated to a client. An update is made to the entry when the port is no longer used by the client. 

      CREATE TABLE `afwallocation` (
  `afwallocation_id` int(10) unsigned NOT NULL auto_increment,
  `account_id` int(10) unsigned default NULL,
  `afwserver_id` int(10) unsigned NOT NULL default '0',
  `clientip` int(10) unsigned NOT NULL default '0',
  `clientport` int(10) unsigned NOT NULL default '0',
  `allocatedip` int(10) unsigned default NULL,
  `allocatedport` int(10) unsigned default NULL,
  `clientprotocol` varchar(3) default NULL,
  `peerprotocol` varchar(3) default NULL,
  `starttime` datetime default '1970-01-01 00:00:00',
  `endtime` datetime default NULL,
  `bandwidth` int(10) unsigned default NULL,
  `status` varchar(15) default 'UNSPECIFIED',
  PRIMARY KEY  (`afwallocation_id`),
  KEY `allocation_index_idx` (`allocatedip`,`allocatedport`,`starttime`),
  KEY `starttime_index_idx` (`starttime`),
  KEY `update_index_idx` (`endtime`,`afwserver_id`),
  KEY `account_time_index_idx` (`account_id`,`starttime`)
)
      Column Type Description
      afwallocation_id integer(10) A unique identifier for the port allocation
      account_id integer(10) A unique identifier for the user account
      afwserver_id integer(10) A unique identifier for the AnyFirewall Server
      clientip integer(10) The IP address of the client
      clientport integer(10) The port of the client
      allocatedip integer(10) The IP address of the allocation on the AnyFirewall Server
      allocatedport integer(10) The port of the allocation on the AnyFirewall Server
      clientprotocol varchar(3) The protocol used by the client (UDP, TCP, TLS)
      peerprotocol varchar(3) The protocol used by the peer (UDP, TCP, TLS)
      starttime datetime The date/time that this allocation entry is first effective
      endtime datetime The date/time that this allocation entry is no longer effective
      bandwidth integer(10) The bandwidth allowed on this allocation
      status varchar(3) The status of this firewall allocation. Possible values are TCPRELAY, UNSPECIFIED, P2P, HTTPRELAY, UDPRELAY

      afwstatistics

      The server periodically logs its statistics. The default period is 15 minutes and it is configurable in the server configuration file (see Configuration).

      CREATE TABLE `afwstatistics` (
  `afwstatistics_id` int(10) unsigned NOT NULL auto_increment,
  `recordtime` datetime NOT NULL default '1970-01-01 00:00:00',
  `afwserver_id` int(10) unsigned NOT NULL default '0',
  `sessions` int(10) unsigned NOT NULL default '0',
  `sessionmax` int(10) unsigned NOT NULL default '0',
  `sessionstarted` int(10) unsigned NOT NULL default '0',
  `sessionended` int(10) unsigned NOT NULL default '0',
  `peakallocated` int(10) unsigned NOT NULL default '0',
  `peakallocatedmax` int(10) unsigned NOT NULL default '0',
  `bytestoclient` int(10) unsigned NOT NULL default '0',
  `bytesfromclient` int(10) unsigned NOT NULL default '0',
  `bytestopeer` int(10) unsigned NOT NULL default '0',
  `bytesfrompeer` int(10) unsigned NOT NULL default '0',
  `writeinterval` int(10) unsigned NOT NULL default '0',
  PRIMARY KEY  (`afwstatistics_id`),
  KEY `record_server_index_idx` (`recordtime`,`afwserver_id`)
)
      Column Type Description
      afwstatistics_id integer(10) A unique identifier for the afwstatistics record
      recordtime datetime The date/time that this record was created
      afwserver_id integer(10) A unique identifier for the AnyFirewall Server
      sessions integer(10) The number of sessions recorded during the interval
      sessionmax integer(10) The maximum number of concurrent sessions
      sessionstarted integer(10) The number of sessions started during this period
      sessionended integer(10) The number of sessions ended during this period
      peakallocated integer(10) The maximum number of concurrent allocations
      peakallocatedmax integer(10) The maximum number of concurrent allocations allowed by license server
      bytestoclient integer(10) The total number of bytes sent to clients during the recording interval
      bytesfromclient integer(10) The total number of bytes received from clients during the recording interval
      bytestopeer integer(10) The total number of bytes sent to peers during the recording interval
      bytesfrompeer integer(10) The total number of bytes received from peers during the recording interval
      writeinterval integer(10) The interval between statistics being stored on the database (in minutes)

      afwsummary

      In this table, the server stores server specific statistics summarized per day.

      CREATE TABLE `afwsummary` (
  `day` date NOT NULL default '1970-01-01',
  `afwserver_id` int(10) unsigned NOT NULL default '0',
  `sessions` int(10) unsigned NOT NULL default '0',
  `sessionmax` int(10) unsigned NOT NULL default '0',
  `sessionstarted` int(10) unsigned NOT NULL default '0',
  `sessionended` int(10) unsigned NOT NULL default '0',
  `peakallocated` int(10) unsigned NOT NULL default '0',
  `peakallocatedmax` int(10) unsigned NOT NULL default '0',
  `bytestoclient` bigint(20) unsigned NOT NULL default '0',
  `bytesfromclient` bigint(20) unsigned NOT NULL default '0',
  `bytestopeer` bigint(20) unsigned NOT NULL default '0',
  `bytesfrompeer` bigint(20) unsigned NOT NULL default '0',
  `writeinterval` int(10) unsigned NOT NULL default '0',
  PRIMARY KEY  (`day`,`afwserver_id`),
  KEY `afw_summary_serv_index_idx` (`afwserver_id`,`day`)
)
      Column Type Description
      day date The day summarized in this record of statistics
      afwserver_id integer(10) A unique identifier for the AnyFirewall Server
      sessions integer(10) The number of sessions recorded during the day
      sessionmax integer(10) The maximum number of concurrent sessions
      sessionstarted integer(10) The number of sessions started during this period
      sessionended integer(10) The number of sessions ended during this period
      peakallocated integer(10) The maximum number of concurrent allocations
      peakallocatedmax integer(10) The maximum number of concurrent allocations allowed by license server
      bytestoclient Big integer(20) The total number of bytes sent to clients during the day
      bytesfromclient Big integer(20) The total number of bytes received from clients during the day
      bytestopeer Big integer(20) The total number of bytes sent to peers during the day
      bytesfrompeer Big integer(20) The total number of bytes received from peers during the day
      writeinterval integer(10) The server increments this value each time a record is written

      afwuserstatistics

      In this table, the server stores user specific statistics summarized per day.

      CREATE TABLE `afwuserstatistics` (
  `account_id` int(10) unsigned NOT NULL default '0',
  `day` date NOT NULL default '1970-01-01',
  `bytestopeer` int(10) unsigned NOT NULL default '0',
  `bytestoclient` int(10) unsigned NOT NULL default '0',
  `bytesfrompeer` int(10) unsigned NOT NULL default '0',
  `bytesfromclient` int(10) unsigned NOT NULL default '0',
  PRIMARY KEY  (`account_id`,`day`)
)
      Column Type Description
      account_id integer(10) A unique identifier for the user account
      day date The day summarized in this record of statistics for the specific user
      bytestopeer integer(10) The total number of bytes sent to peers by the specified user during the day
      bytestoclient integer(10) The total number of bytes sent to clients by the specified user during the day
      bytesfrompeer integer(10) The total number of bytes received from peers by the specified user during the day
      bytesfromclient integer(10) The total number of bytes received from clients by the specified user during the day

      afwsessionstats

      This table is used by The Eyeball AnyFirewall™ for storing session information to the database. Call completion type (P2P or Relay), call start time and duration is stored in this table.

      CREATE TABLE `afwsessionstats` (
  `afwsessionstats_id` int(10) unsigned NOT NULL auto_increment,
  `afwserver_id` int(10) unsigned NOT NULL default '0',
  `user_id` varchar(512) default 'UNSPECIFIED',
  `strsessionidentifier` varchar(20) NOT NULL default '0',
  `starttime` int(15) NOT NULL default '0',
  `callcompletiontype` int(10) unsigned NOT NULL default '0',
  `callduration` int(10) unsigned NOT NULL default '0',
  PRIMARY KEY  (`afwsessionstats_id`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8;
      Column Type Description
      afwsessionstats_id integer(10) Unique identifier for the session status
      afwserver_id integer(10) Unique identifier for the AnyFirewall Server
      user_id varchar(512) The user ID associated with this account
      strsessionidentifier varchar(20) Randomly generated value by AFE to identify each session
      starttime integer(15) Call start time
      callcompletiontype integer(10) Call completion type (P2P=1/Relay=2/TCP-Relay=3/HTTP-Relay=4)
      callduration integer(10) Duration of the call
